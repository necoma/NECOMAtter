NECOMATter REST interface manual

get User tweets

URL: /user/[user_name].json
method: GET
get user tweets.
options limit: It limits the number of a paid value.. since_time: This option intends for the past at appointed time.
result data format: [{"text": tweet_text, "time": formatted time string, "unix_time": unix time[second]}, ...]
example: /user/iimura.json?limit=10&since_time=1385021149.0

get User timeline

URL: /timeline/[user_name].json
method: GET
get user timeline.
options limit: It limits the number of a paid value.. since_time: This option intends for the past at appointed time.
result data format: [{"text": tweet_text, "time": formatted time string, "user": user name, "unix_time": unix time[second]}, ...]
example: /timeline/iimura.json?limit=10&since_time=1385021149.0

get User list

URL: /user_name_list.json
method: GET
get all user name list.
result data format: [user name, ...]

post tweet

URL: /post.json
method: POST
post new tweet. (required sign in)
post data format: {"text": tweet text }
result data format: {"result": 'ok' or 'error', 'description': description text}

follow user

URL: /follow.json
method: POST
follow user (required sign in)
post data format: {"user_name": target user name}
result data format: {"result": 'ok' or 'error', 'description': description text}

unfollow user

URL: /unfollow.json
method: DELETE
unfollow user (required sign in)
post data format: {"user_name": target user name}
result data format: {"result": 'ok' or 'error', 'description': description text}

unfollow user(altanative for DELETE method unsupported client)

URL: /unfollow.json
method: POST
unfollow user (required sign in)
post data format: {"user_name": target user name}
result data format: {"result": 'ok' or 'error', 'description': description text}

get followed user name list

URL: /user/[user_name]/followed_user_name_list.json
method: GET
I acquire the list of the user where [user_name] does follow.
result data format: ['user_name', ...]
