NECOMAtter REST interface manual

get User tweets

URL: /user/[user_name].json
method: GET
get user tweets.
options limit: It limits the number of a paid value.. since_time: This option intends for the past at appointed time.
result data format: [{"text": tweet_text, "time": formatted time string, "unix_time": unix time[second], "user_name": user name, "id": this tweet's TweetID}, ...]
example: /user/iimura.json?limit=10&since_time=1385021149.0

get User list

URL: /user_name_list.json
method: GET
get all user name list.
result data format: [user name, ...]

post tweet

URL: /post.json
method: POST
post new tweet. (required sign in)
post data format: {"text": tweet text, "api_key": API key, "user_name": user name }
result data format: {"result": 'ok' or 'error', 'description': description text}

URL: /follow.json

post data format: {"user_name": target user name, "api_key": API key}
result data format: {"result": 'ok' or 'error', 'description': description text}

create new API key

URL: /user_setting/create_new_key.json
method: PUT(recomended), POST
create new API key (required sign in)
no option.
result data format: {"result": 'ok', "key": created api key string}

delete user API key

URL: /user_setting/key/.json
method: DELETE(recomended), POST
delete API key (required sign in)
post data format: {"user_name": target user name, "api_key": API key}
result data format: {"result": 'ok'}

streaming read whole timeline with regular expression match

URL: /stream/regexp.json
method: POST
(required sign in)
post data format: {"regexp": regular expression string(Python format), "user_name": target user name, "api_key": API key}
result data format: one tweet in one line like: {"text": tweet_text, "time": formatted time string, "unix_time": unix time[second], "user_name": tweet user name, "id": this tweet's TweetID, "match_result": [match result string (if you use '()' in regexp option), ...]}
example post data: {"regexp": "(\d+.\d+.\d+.\d+)", "user_name": "iimura", "api_key": "...."}
