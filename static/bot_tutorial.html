How to create NECOMATter BOT

When you make BOT in NECOMATter, it is necessary to perform two of the making of the BOT-user and the generation of API-KEY. I perform the making of the BOT-user from the sign up page. When you make a user and enter the signature in NECOMATter, a page of the API-KEY setting page is available. Please make API-KEY in this page.
There is two kinds of BOT of NECOMATter.
  • POST BOT
  • STREAM BOT
POST BOT is BOT to perform tweet. STREAM BOT is BOT to watch tweet. In many cases, STREAM BOT will have the function of the POST.

POST BOT

POST_BOT performs tweet by performing a POST of the data of the JSON form for the following URLs.
  • REST URL: /post.json
  • HTTP METHOD: POST
  • required header: Content-Type: application/json; charset=utf-8
  • POST data: JSON { "user": "user name string", "api_key": "API-KEY string", "text": "tweet text" }

STREAM BOT

STREAM_BOT is to perform a POST of the data of the JSON form for the following URLs and can watch tweet stream.
  • REST URL: /stream/regexp.json
  • HTTP METHOD: POST
  • required header: Content-Type: application/json; charset=utf-8
  • POST data: JSON { "user": "user name string", "api_key": "API-KEY string", "regexp": "regular expression", "description": "BOT description text" }
